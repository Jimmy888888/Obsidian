- #Golang專案
- ## PoE排程
	- 需求描述
		- 不能跟原有功能衝突
			- 原有功能：總是on / 總是off
		- 新增功能：讓使用者可以自訂每個網孔，一個禮拜中什麼時候通電 ，一天之中可以設定多個時段通電， 以一小時為最小單位
		- 例如：可以設定網孔每個禮拜三09:00~18:00跟禮拜五10:00~23:00通電  其餘時間不通電
	- 挑戰
		- 設計儲存config JSON格式
			- 每個網孔都有一個欄位，欄位包含：
				- 網孔當前PoE狀態(on/off)
				- 網孔PoE排程是否開啟
				- 網孔排程的通電時段
		- Firmware提供的RPC只有on/off  無法計算持續多長時間
	- 解法
		- Backend執行檔啟動時(使用者按下開機鍵)
			- 1.會用Go JSON套件先檢查PoE config是否存在
				- 若不存在 為每個網孔生成預設config
				- 若存在，由於Firmware開機預設PoE是off
					- 1.先檢查網孔PoE排程是否開啟
						- 是，將網孔PoE狀態設為off
						- 否，continue
					- 2.call Firmware RPC開啟PoE狀態為on的網孔
			- 2.生出一個Goroutine 每分鐘根據config內容執行相對應操作
		- 新增行程，在對應的網孔config寫入行程，需判斷有無跟已經存在的行程衝突 如果有就無法新增
		- 刪除行程，根據API input data，刪除對應的網孔config
		- 執行行程，每分鐘執行一次
			- ```
			  for {
			      poeSched()
			      time.Sleep(60* time.Second)
			  }
			  ```
			- poeSched()會先取得系統目前時間，根據每個網孔config做相對應操作
				- 先檢查網孔PoE排程是否開啟
					- 是，根據系統時間，對比config時段
						- 系統時間在時段內，對比網孔PoE狀態
							- 若網孔PoE狀態是off，call Firmware RPC使PoE on
							- 若網孔PoE狀態是on，continue
						- 系統時間在時段外，對比網孔PoE狀態
							- 若網孔PoE狀態是off，continue
							- 若網孔PoE狀態是on，call Firmware RPC使PoE off
					- 否，continue
