- ### C++的Web API框架特點
	- 這些框架通常具備以下特點：
	- **高性能：** C++本身的性能優勢，加上這些框架對底層的優化，使得它們在處理高並發請求時表現出色。
	- **易用性：** 提供直觀的API和豐富的特性，降低開發者的學習成本，提高開發效率。
	- **可擴展性：** 支持自定義中間件、插件等，方便開發者根據需求進行擴展。
	- **安全性：** 提供一系列安全機制，如防止SQL注入、跨站腳本攻擊等。
- ### 常見的C++ Web框架
	- **Oatpp:**
		- 輕量級、高性能，適合嵌入式系統和微服務。
		- 支持WebSocket、REST、微服務、异步请求。
		- 跨平台，可移植性強。
	- **uWebSockets:**
		- 極輕量級、高性能的WebSocket服务器库。
		- 內建Bun，性能極佳的JavaScript運行時。
	- **Poco:**
		- 綜合性庫，包含網絡服务器、数据库、加密等多種功能。
		- 跨平台，穩定可靠。
	- **Crow:**
		- 微型Web框架，簡單易用。
	- **CppCMS:**
		- 功能齊全的Web框架，支持MVC模式、模板引擎等。
- ### 選擇框架的建議
	- 選擇適合的C++ Web框架，需要考慮以下因素：
	- **項目規模和複雜度：** 小規模項目可能更適合輕量級的框架，而大型項目則需要功能更全面的框架。
	- **性能要求：** 如果對性能要求很高，可以選擇uWebSockets或Oatpp。
	- **開發團隊的經驗：** 如果團隊對C++模板技術熟悉，CppCMS可能是一個不錯的選擇。
	- **社區支持：** 選擇一個活躍的社區，有助於解決開發過程中遇到的問題。
- ### 綜合比較
	- | 框架 | 特點 | 適合場景 |
	  | Oatpp | 輕量級、高性能、跨平台 | 嵌入式系統、微服務 |
	  | uWebSockets | 極輕量級、高性能WebSocket | 实时应用 |
	  | Poco | 綜合性庫，功能全面 | 大型项目 |
	  | Crow | 微型框架，简单易用 | 小型项目 |
	  | CppCMS | 功能齊全，支持MVC | 中大型项目 |
- ### 總結
	- C++擁有多種優秀的Web框架，選擇哪個框架取決於你的具體需求。建議你在選擇之前，先對這些框架進行深入了解，並結合你的項目特點做出最優的選擇。
