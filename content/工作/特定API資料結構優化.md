- #[[C++專案]]
-
- ## 需求
	- 提升特定功能API的吞吐量
- ## 問題描述
	- 此功能讓使用者透過 新增POST/修改PUT/排序優先級PUT/刪除DELET 
	  具有固定格式的規則，達到控制特定功能的效果
	- 每次使用者透過前端編輯 **一條規則字串** 內容後(新增POST/修改PUT)
	- 前端會即時自動將 **所有規則字串** 內容傳回後端
	- 後端必須對 **所有規則字串** 做格式檢查，再檢查所有規則沒有互相衝突
	- 確認沒問題才會以JSON格式照優先級順序儲存在config，並回傳內容給前端
	- 假設有n個規則，後端檢查所有規則格式 ＆ 確認所有規則之間沒有互相衝突，時間複雜度約為O(n^2) ，
	  更精確來說是n + n * (n + 1) / 2
	- 時間複雜度約為O(n^2) ，當規則數量增多，使用者可以明顯感受到API回應變慢
		- 排序優先級PUT API，雖然實際規則字串沒變，只有改變規則之間的順序，runtime仍跟
		  新增POST/修改PUT 一樣是O(n^2)
- ## 解法
	- 需要減少後端執行 規則格式檢查 ＆ 規則之間的衝突檢查
	- ### 1.在後端用編號代替實際字串
		- 1-1.建立index，為每條規則建立unique編號
			- 每條規則在JSON array ArrayTXT 中的順序就是它的編號
			- 新增另一個JSON array ArrayID，編號在array中的順序就是實際的規則優先級順序
		- 1-2.新增POST 一條規則時
			- 前端會回傳完整的一條規則內容，後端只對那一條規則做格式檢查 ＆ 確認跟其他規則有無衝突
			  ，時間複雜度約為O(n)
			- 完成確認後，寫入config
				- 在ArrayTXT最後面加上新增規則
				- 根據新增規則在ArrayTXT中的順序，在ArrayID最後面新增編號
		- 1-3.修改PUT 一條規則時
			- runtime同 新增POST 一條規則，不過只寫入ArrayTXT
	- ### 2.排序規則優先級
		- 使用JSON array ArrayID ，參考1-1
		- 2-1.當規則優先級排序時，前端會回傳被移動的那條規則的：
			- 移動後的優先順序
			- 移動前的優先順序
			- 編號
		- 2-2.後端只需對ArrayID做順序移動，時間複雜度為O(1)
	- ### 3.刪除規則
		- 3-1.當前端刪除一條規則，會回傳被刪除規則的編號
		- 3-2.後端會根據編號
			- 刪除ArrayTXT(參考1-1)中的規則
			- 刪除ArrayID中的編號，並做順序移動
